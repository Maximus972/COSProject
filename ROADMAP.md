# Дорожная карта ContraOS

Маршрут развития ядра, выстроенный по зависимостям: каждый этап опирается на
предыдущий. Цель проекта — учебный долгосрочный хобби-OS, где важно понимание
архитектуры, а не скорость. После Этапа 4 сознательно выбран порядок
**userspace раньше диска**.

## Текущее состояние (стартовая точка)

Boot через GRUB multiboot, GDT, IDT + PIC remap, обработка исключений CPU,
драйверы клавиатуры и таймера (PIT 100 Гц), VGA text-mode, цепочка
terminal → console → shell (команды `help`, `clear`). Ядро 32-битное.

---

## Этап 0 — Подготовка фундамента

Мелкие, но обязательные вещи до работы с памятью.

- Передать ядру указатель на multiboot info от GRUB (сейчас он теряется).
- Мини-libc: `memset`, `memcpy`, `memmove`, `strlen`, `strcmp`, `memcmp`,
  и вывод чисел (`itoa` / hex).
- Скроллинг терминала вместо перескока на строку 0; опционально аппаратный курсор.

**Веха:** shell печатает карту памяти от GRUB, длинный вывод корректно скроллится.

## Этап 1 — Физический менеджер памяти (PMM)

- Парсинг карты памяти из multiboot info (доступные регионы RAM).
- Bitmap-аллокатор фреймов по 4 КБ; занятыми помечены ядро и недоступные области.

**Веха:** `pmm_alloc_frame()` выдаёт уникальные адреса, `pmm_free_frame()`
возвращает их в оборот, есть учёт used/free.

## Этап 2 — Paging (виртуальная память)

- Page directory и page tables, identity-mapping ядра и видеопамяти.
- Включение paging; обработчик page fault (#14) с чтением `cr2`.

**Веха:** ядро работает с включённым paging, обращение к немапленному адресу
даёт осмысленный отчёт, а не тройную перезагрузку.

## Этап 3 — Куча ядра

- `kmalloc` / `kfree` поверх paging: сначала bump-allocator, затем free-list
  с объединением соседних блоков.

**Веха:** многократные `kmalloc`/`kfree` без утечек и порчи памяти.

## Этап 4 — Многозадачность ядра

- Структура задачи, переключение контекста на ассемблере.
- Round-robin планировщик, вызываемый из обработчика таймера.

**Веха:** две kernel-задачи печатают по очереди без ручного переключения.

## Этап 5 — Системные вызовы

- Шлюз `int 0x80`, диспетчеризация по номеру вызова.
- Первые вызовы: `write`, `exit`, `yield`.

**Веха:** задача выводит на экран только через syscall, без прямого доступа к VGA.

## Этап 6 — User mode (ring 3)

- TSS, записи GDT для user code/data, переход в ring 3 через `iret`.
- Защита памяти ядра от пользовательского кода флагами страниц.

**Веха:** программа в ring 3 нарушает доступ → ядро её ловит, а не виснет.

## Этап 7 — Диск и файловая система

- Драйвер ATA PIO (чтение/запись секторов).
- Сначала initrd (модуль GRUB), затем при желании FAT.

**Веха:** команды shell `ls` / `cat` читают файлы.

## Этап 8 — Загрузчик программ (ELF)

- Парсинг ELF, маппинг сегментов в адресное пространство задачи, старт в ring 3.

**Веха:** `exec hello` грузит отдельный бинарник с диска и запускает в user mode.

---

## Этап 9+ — Расширения

Больше syscalls, userspace-libc, pipes между процессами, дополнительные драйверы
(serial для логов, RTC, мышь), в перспективе переход в long mode (64 бита).
